description: "The three commands owned by the @vendoai/vendo package: init, doctor, and sync."
---

The umbrella owns the `vendo` bin. It has three commands.

## `vendo init`

Interactive setup. It scans the app, interviews you about risk labels, theme,
and remix candidates, then proposes the handler and `<VendoRoot>` wiring. Every
code change is permission-gated and shown as a diff. Answers are written to
`.vendo/overrides.json` and respected by future sync runs.

Init detects the host framework and scaffolds only what applies. Next.js apps
get an `app/api/vendo/[...vendo]/route.ts` and a layout that wraps
`<VendoRoot>`. Express (and other non-Next) apps get a `vendo/server.ts`
containing the `createVendo()` composition and a Node HTTP↔fetch adapter, plus
a starter `vendo/ai.ts` — mount the adapter with
`app.use("/api/vendo", mountVendo())` and wrap the client entry in
`<VendoRoot>` to finish.

`--agent` emits the setup plan, writes nothing, and asks at most three
plain-language questions.

## `vendo doctor`

Checks wiring and makes one live `/status` round trip. It recognizes both
Next.js (catch-all route + layout `<VendoRoot>`) and Express (server uses
`createVendo` from `@vendoai/vendo/server`; client mounts `<VendoRoot>`), and
ends with the config line that unlocks each remaining block.

Exit codes:

- `0`: green
- `1`: broken wiring

## `vendo sync`

Runs the build-step extraction manually. It updates tool extraction and remix
baselines. The default is fail-soft and warns on breaking changes.

Exit codes:

- `0`: success, including fail-soft warnings
- `2`: breaking changes when `--strict` is set

## Telemetry opt-out

Set `VENDO_TELEMETRY_DISABLED=1` or set `"optedOut": true` in
`~/.vendo/telemetry.json`. `DO_NOT_TRACK`, CI, and production runtime also
disable build and development telemetry.
