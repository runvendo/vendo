## 1. Install and init

```bash
npm install @vendoai/vendo
npx vendo init
```

Init scaffolds the wire route, turns your host actions into agent tools, and
captures your theme ([what init writes](/connect/vendo-init)). Lift the
`createVendo` call init wrote into a shared module, so your chat route can
use the same instance:

```ts
// lib/vendo.ts — the call init wrote, moved out of the wire route
import { createVendo } from "@vendoai/vendo/server";

export const vendo = createVendo({
  // …the options init wrote (catalog, policy, …)
  principal, // who your callers act as — from your auth
});
```

`principal` comes from your auth
([presets for Auth.js, Supabase, Clerk, Auth0](/connect/act-as-presets)).

```ts
// app/api/vendo/[...vendo]/route.ts — now just the handler
import { nextVendoHandler } from "@vendoai/vendo/server";
import { vendo } from "@/lib/vendo";

export const { GET, POST, PUT, PATCH, DELETE } = nextVendoHandler(vendo);
```

One `next.config.ts` line keeps Vendo's native dependencies out of the
bundler:

```ts
const nextConfig = { serverExternalPackages: ["esbuild", "@electric-sql/pglite"] };
```

## 2. Spread the pack

```ts
// app/api/chat/route.ts — your existing route, one spread
import { vendoTools } from "@vendoai/vendo/ai-sdk";
import { vendo } from "@/lib/vendo";

const result = streamText({
  model,
  messages,
  tools: { ...yourTools, ...(await vendoTools(vendo, { principal })) },
});
```

That one spread gives your agent a guarded `vendo_<action>` tool per host
action, `vendo_create_app` (micro-apps in your chat), and `vendo_delegate`.
`principal` is the same caller your auth resolves in `lib/vendo.ts`
([why](/existing-agents/configuration#one-principal-two-places)).

## 3. Render the embeds

Wrap the chat once in `VendoProvider` and hand finished tool outputs to
`<VendoToolResult>`. In `useChat`, `vendo_*` tools stream as `dynamic-tool`
message parts:

```tsx
// app/page.tsx — inside your existing message-parts loop
case "dynamic-tool":
  return part.state === "output-available"
    ? <VendoToolResult output={part.output} />
    : <div>Running {part.toolName}…</div>;
```

## Try it

1. **"What's the weather in Paris?"** — a guarded read; your model narrates
   the result.
2. **"Make me a dashboard comparing the weather in Paris, London and
   Tokyo."** — a micro-app builds live, inline in your chat.
3. **Ask for a destructive action** from `.vendo/tools.json` in plain
   language — it parks as an approval card; Approve runs it in place.

Your loop keeps its own model key; Vendo's app builds resolve theirs through
the [dev-mode credential ladder](/connect/dev-mode). No key? `npx vendo
login` mints a free one. Details and knobs:
[Configuration](/existing-agents/configuration).
