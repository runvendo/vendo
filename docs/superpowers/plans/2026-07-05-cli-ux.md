# Vendo CLI UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> Per Yousef's global rules this plan is high-level: goals, files, decisions, and verification per task; no code. The spec is `docs/superpowers/specs/2026-07-05-cli-ux-design.md` and is the source of truth for behavior.

**Goal:** Rebuild the Vendo CLI around the three-tier story (init sets up, refresh catches up, sync silently maintains), with an interactive wizard, additive re-runs, a doctor command, and one shared output language.

**Architecture:** All work lives in `packages/vendo-cli` plus docs. A new renderer module (`ui.ts`) and prompt module (`interact.ts`) are the only new UX infrastructure; every command renders through them. Init/refresh share one additive pipeline driven by a `.vendo` state-inspection module. Remix anchors follow the existing never-guess codemod contract (LLM chooses targets, deterministic AST splice applies them).

**Tech stack:** TypeScript, vite-bundled bin, vitest, TypeScript compiler API for splices, `ai` SDK with MockLanguageModelV3 in tests, picocolors + @clack/prompts (new deps, inlined by the bundle).

**Working rules for every task:** TDD (write the failing test first, watch it fail, implement, watch it pass), one commit per task minimum, `pnpm --filter @vendoai/cli test` green before each commit. Follow existing patterns: injectable io/log/model seams, fail-open extraction, fail-closed annotations.

## Decisions locked here (so tasks don't relitigate)
- Renderer owns all TTY/CI/NO_COLOR detection; commands never branch on environment.
- Pickers and prompts are seams, not direct clack calls, so every test runs headless.
- LLM proposes, deterministic code applies: both pickers' generation and the anchor splice never trust LLM output without validation (schema rescue, syntax check, exclusion lists).
- `refresh` is the documented growth verb; `init` on a wired app silently behaves as refresh rather than erroring or renaming itself.
- Sync gains no LLM, no suggestions, no new responsibilities — output changes only.

## Out of scope (from spec)
Real publish (ENG-198), remix context inference, LLM in sync, prompts beyond key + two pickers, in-app pickers/badges.
