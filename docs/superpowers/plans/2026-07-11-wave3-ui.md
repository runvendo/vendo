# Wave 3 — @vendoai/ui Implementation Plan

**Goal:** Build `@vendoai/ui` fresh against the frozen contract (docs/contracts/08-ui.md): headless client + hooks, chrome surfaces, the `vendo-genui/v1` tree renderer, and the voice subpath — one package, subpath exports, depending on `@vendoai/core` only.

**Execution model:** Fable orchestrates; Codex implements via exec lanes (no network in lanes; Fable runs installs and commits). One PR to main, dual-reviewed, root gates green, browser screenshots for all UI surfaces.

## Shape

One package `packages/ui`, four entries:

- `@vendoai/ui` — `createVendoClient` (typed fetch/SSE bindings for every 09 §3 wire route), `VendoProvider`, the ten hooks (08 §3), theme resolution.
- `@vendoai/ui/chrome` — the ten shipped surfaces (08 §4), styled exclusively from `VendoTheme` CSS variables, WCAG 2.1 AA + keyboard-complete.
- `@vendoai/ui/tree` — `TreeView` (format-dispatching registry; `vendo-genui/v1` renderer), prewired primitives, `$path`/`$state` bindings, generated-component iframe jail, containment, skeletons, `AppFrame`, pin mounting with fallback.
- `@vendoai/ui/voice` — voice stage driver seam + `useVoice` + the WebRTC realtime driver.

## Decisions (flagged for Yousef's review at PR)

1. **Dependencies:** peers = `react`/`react-dom` ≥ 18 and `ai` ≥ 6 (singleton rule from 00 conventions); `@ai-sdk/react` is a regular dependency (implementation detail of `useVendoThread`, which must be useChat-compatible and expose `addToolApprovalResponse`). Block-layering ("depends on core only") governs `@vendoai/*` edges, which stay core-only.
2. **Voice seam is driver-injection.** The frozen wire table (09 §3) has no voice route; the legacy pattern (host backend mints an ephemeral credential, browser connects WebRTC directly) is the only freeze-compatible shape. `/voice` exports a `VoiceDriver` interface plus an OpenAI-Realtime driver taking `getSession()`; hosts pass a driver to `VendoStage`/`useVoice`.
3. **Theme:** resolved theme = built-in defaults ⊕ `VendoProvider.theme` prop. The "server-extracted theme" arrives via the umbrella's `<VendoRoot>` (09 §1) passing `.vendo/theme.json` in as that prop — no theme wire route exists.
4. **`$state` persistence:** the renderer keeps view state client-side and exposes a change chokepoint; per-user-per-app persistence rides the state singleton owned by apps/umbrella (06 §6), not a ui-invented wire route.
5. **Jail mechanics:** transplant the legacy stage pattern (sandboxed iframe, strict CSP `connect-src 'none'`, blob-URL ESM import, postMessage bridge: props in / actions out) rewritten for the frozen `Tree` shape; JSX in generated sources is compiled in-jail (sucrase).
6. **Browser tests:** Playwright + axe live in `packages/ui`, run as a separate script (not turbo `test` — CI runs deterministic doubles per the testing doctrine); Fable runs them locally and attaches screenshots to the PR.

## Testing doctrine mapping (binding)

- Hooks + client: e2e against an in-test HTTP wire server speaking the 09 §3 route table exactly (routes, envelopes, SSE ai-SDK UI message stream, error envelope + status map). No internals-testing.
- Renderer + chrome: real browser (Playwright) with screenshots; WCAG/keyboard assertions on chrome (axe + tab-order tests).
- Jail security assertions in-browser: generated code cannot fetch (CSP), cannot touch the parent DOM; an erroring node/component stays contained.

## Lanes

- **Lane 0 (Fable):** package scaffold — manifest with subpath exports, tsconfig, dep declarations, shared context/types stubs; install; commit. Unblocks all lanes.
- **Lane A (Codex):** client + provider + hooks + wire-server tests.
- **Lane B (Codex):** tree renderer subpath + jail + renderer unit/jsdom tests.
- **Lane C (Codex):** chrome surfaces + theme CSS variables + a11y structure.
- **Lane D (Codex):** voice subpath.
- **Lane E (Codex, after A–C):** Playwright harness + browser/a11y tests; Fable executes browsers and captures screenshots.

Checkpoints after each lane merge: review diff, run gates, re-merge `yousefh409/v0-wave3-core` as it evolves, update the Orca worktree comment.
