# Invoice Kanban Board

A full web application for managing invoices using a drag-and-drop kanban board interface.

## Features

- **Three Status Columns**: Draft, Sent, and Paid
- **Drag and Drop**: Move invoice cards between columns to update their status
- **Real-time Updates**: Status changes are immediately persisted to the server
- **Responsive Design**: Works on desktop and mobile devices
- **Theme Support**: Accepts optional `vendoTheme` query parameter for host branding

## API Endpoints

### POST /fn/listInvoices
Returns all invoices.

**Response:**
```json
{
  "result": {
    "invoices": [...]
  }
}
```

### POST /fn/updateInvoiceStatus
Updates an invoice's status.

**Request:**
```json
{
  "args": {
    "id": "inv-001",
    "status": "sent"
  }
}
```

**Response:**
```json
{
  "result": {
    "invoice": {...}
  }
}
```

### POST /fn/getInvoice
Retrieves a single invoice by ID.

**Request:**
```json
{
  "args": {
    "id": "inv-001"
  }
}
```

**Response:**
```json
{
  "result": {
    "invoice": {...}
  }
}
```

The app is seeded with 6 sample invoices:
- 2 Draft invoices (Acme Corp, Tech Solutions Inc)
- 2 Sent invoices (Global Industries, StartUp Labs)
- 2 Paid invoices (Enterprise Co, Mega Systems)

## Usage

1. Access the web interface at `http://localhost:$PORT/`
2. Drag invoice cards between columns to change their status
3. The changes are automatically saved to the server
4. Refresh the page to see the persisted state

## Architecture

- **Zero Dependencies**: Pure Node.js with no external packages
- **In-Memory Storage**: Invoice data stored in process memory
- **Drag & Drop API**: Native HTML5 drag and drop
- **RESTful Functions**: Clean separation between UI and business logic
